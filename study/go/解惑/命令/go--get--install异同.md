`go get` 和 `go install` 是Go工具链中的两个常用命令，它们在获取和安装Go包和工具时起不同的作用。以下是它们的主要异同：

### `go get`

#### 功能
- 下载并安装指定的Go包及其依赖。
- 如果包中包含可执行文件，则会编译并安装这些可执行文件到 `GOPATH/bin` 目录中。
- 支持从版本控制系统（如Git、Mercurial、Subversion等）获取远程代码库。

#### 主要特点
- **下载源码**：将包的源代码下载到 `GOPATH/src` 目录中。
- **安装可执行文件**：如果包中包含命令行工具（main包），则会编译并安装到 `GOPATH/bin` 目录中。
- **自动处理依赖**：会自动下载并安装包的所有依赖。
- **模块支持**：在使用Go模块（`go.mod`）时，`go get` 会更新 `go.mod` 和 `go.sum` 文件，并下载依赖到模块缓存中。

#### 示例
```sh
go get github.com/user/repo  # 下载并安装 repo 包及其依赖
go get -u github.com/user/repo  # 更新 repo 包及其依赖
```

### `go install`

#### 功能
- 编译并安装指定的Go包或命令。
- 将编译后的可执行文件安装到 `GOPATH/bin` 目录中。

#### 主要特点
- **编译和安装**：编译指定的包并将生成的二进制文件安装到 `GOPATH/bin` 目录中。
- **不下载源码**：如果包的源代码尚未下载，`go install` 不会下载它，只能在本地源码存在时使用。
- **仅用于编译包**：主要用于本地开发过程中，将包编译并安装为可执行文件。

#### 示例
```sh
cd $GOPATH/src/github.com/user/repo
go install ./...  # 编译并安装 repo 包中的所有子包
```

### 异同点总结

| **命令**     | **功能**                                           | **主要作用**                                 | **是否下载源码** | **是否安装可执行文件**   |
| ------------ | -------------------------------------------------- | -------------------------------------------- | ---------------- | ------------------------ |
| `go get`     | 下载并安装指定的Go包及其依赖，更新依赖时也可使用。 | 获取和更新远程包及其依赖。                   | 是               | 是                       |
| `go install` | 编译并安装指定的Go包或命令。                       | 本地开发过程中，将包编译并安装为可执行文件。 | 否               | 是（仅在本地已有源码时） |

### 使用建议

- **`go get`**：适用于获取和更新远程代码库及其依赖，尤其是初次下载某个包或更新包时使用。
- **`go install`**：适用于本地开发过程中，将已存在于 `GOPATH/src` 或当前模块中的包编译并安装为可执行文件。
