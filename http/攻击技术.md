- 概述

```tex
1、在HTTP请求报文内容加载攻击代码，旧能发起对Web应用的攻击。通过URL查询字段或表单、HTTP首部、Cookie等途径把攻击代码传入，若这时Web应用存在安全漏洞，那内部信息就会遭到窃取，或者被攻击者拿到管理权限。
```

- 主动攻击

```tex
主动攻击的代表有 SQL注入 和 OS命令注入 攻击。
```

- 被动攻击

```tex
代表有跨站脚本攻击和跨站点请求伪造
```

- 其他攻击一览

```tex
1、在 HTML 以及脚本等做处理，使用户提交多一份到攻击者那里去；
2、Cookie的攻击，利用脚本获取Cookie进行信息窃取；
3、SQL注入，在字段信息处需改，增加字段等；
4、OS注入，生成os脚本进行攻击；
5、header头部注入攻击；
6、HTTP相应截断攻击，`%0D%0A%0D%0A`字符串可以让文本换行，换行后，自然可以添加更多的头部信息等；
7、根据不同的服务，添加不同的语法，能执行shell的添加shell脚本，如目录攻击等 ?log=../../etc/passwd；
8、针对 include 脚本等添加文件路径等信息；
9、设置或设计上的缺陷：如一些容易被猜测的目录地址、脚本错误抛出的目录信息、以及其他等错误的信息等等；
10、重定向功能的误导，导致用户被欺骗；
11、会话劫持，指攻击者利用某些手段，拿到了用户的会话 ID，从而可以进行交互；
12、跨站请求伪造攻击；
13、密码攻击：穷举法、根据密码字典来对用户密码进行猜测；
14、Dos攻击，是让服务器资源过载，达到服务停止的行为。
```
